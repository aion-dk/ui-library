# Welcome to the Assembly Voting UI Library

This library based on our Design System is meant to help you build `Vue` applications faster.

Here, you can search among our UI components, grab the ones you need, install them on your applications, customize them and you're ready to go. You can also go to the `templates` folder and copy a whole layout.

### Project structure

The Design System and its components follow the principles of <strong>Atomic Design</strong>, where:

- <strong>Atoms</strong> include basic HTML elements and UI components (buttons, inputs, etc.),
- <strong>Molecules</strong> are small groups of atoms working together as a unit (election cards,
  radio groups, etc.),
- <strong>Organisms</strong> are relatively complex UI components composed of groups of atoms and
  molecules (navbar, sidebar, modal, etc.),
- <strong>Templates</strong> include <em>grab-and-go</em> pre-build pages to copy and paste them
  directly inside a project.

<img src={AtomicDesign} />

So the folder structure of the project follows these principles as well.

<div className="subheading">USING THE LIBRARY</div>

<strong>Import the whole library as a plugin (recommended)</strong>

<Source
  language="typescript"
  dark
  format={false}
  code={dedent` 
      {/* main.ts | main.js */}\n
      import * as UILibrary from "@assemblyvoting/electa-ui"; // Imports components
      import "@assemblyvoting/electa-ui/styles"; // Imports styles\n
      const app = createApp(App);\n
      app.use(UILibrary.default); // Plugin-in the library\n
      app.mount("#app");
    `}
/>

<div className="tip-wrapper">
  <span className="tip">Tip</span> You can optionally provide your instance of i18n to the UI
  library on initialization in the second param. In this case, the library will consume the
  translations from the parent application's i18n and will ignore all the internal translations.
</div>

<div className="tip-wrapper">
  <span className="tip">Tip</span> In case you don't pass a i18n instance, you can pass the locale
  as a param to any component that needs to be translated.
</div>

<div className="tip-wrapper">
  <span className="tip important">Warning</span> If that's the case, it is important that you
  provide all translations needed for all of the components you will use.
</div>

<Source
  language="typescript"
  dark
  format={false}
  code={dedent` 
      {/* main.ts | main.js */}\n
      import * as UILibrary from "@assemblyvoting/electa-ui"; // Imports components
      import "@assemblyvoting/electa-ui/styles"; // Imports styles\n
      const app = createApp(App);\n
      app.use(UILibrary.default, i18n); // Plugin-in the library\n
      app.mount("#app");
    `}
/>

The translations provided need to follow this structure:

<Source
  language="typescript"
  dark
  format={false}
  code={dedent` 
      {/* translationFile.es.yml */}\n

        // ... translations
        js:
          components:
            AVComponentName:
              key_1: Translation1
              key_2: Translation2
              key_3: Translation3
        // ... more translations
    `}

/>

<strong>Importing components individually (if you only need a few components)</strong>

<Source
  language="typescript"
  dark
  format={false}
  code={dedent`

      <script>
        import { AVButton } from "@assemblyvoting/electa-ui"
      </script>\n
      <template>
        <AVButton />
      </template>
    `}
/>

  <span className="tip">Tip</span> Check the component both in the storybook view and in the
  development App.

<div className="tip-wrapper">
  <span className="tip">Tip</span> Most of the UI components from our design system uses{" "}
  <a href="https://tailwindcss.com/docs/preflight" target="_blank">
    Tailwindcss utility classes
  </a>
  , you can implement them by using the <code>@apply</code> argument in your regular css file.
  Example:
</div>

<Source
  language="css"
  dark
  format={false}
  code={dedent`
      .my-container {
        @apply utility-class-1 utility-class-2 hover:utility-class-3;
        normal-css-property: blue; 
      }
    `}
/>

<div className="tip-wrapper">
  <span className="tip">Tip</span> The colors we use are named <strong>slate</strong> (avx/us),{" "}
  <strong>gray</strong> (electa), <strong>avblue</strong> and <strong>semantic</strong>. You can
  find the swatches in the <code>Colors</code> page and use them inside the utility classes:{" "}
  <code>bg-avblue-600</code>, <code>text-slate-800</code>, <code>border-semantic-danger-1</code>.
</div>

<div className="tip-wrapper">
  <span className="tip">Tip</span> Remember to use the AV prefix both for component and class names.
</div>

<div className="tip-wrapper">
  <span className="tip">Tip</span> Once the component is ready, you need to include it on the
  component index at <code>@app/src/components.index.ts</code> and then in the Library (both as
  plugin component and as regular export) at <code>@app/src/UILibrary.ts</code>.
</div>
